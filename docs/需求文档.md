# 需求文档（校园生活交友平台）

## 1. 项目概述
参考微博、抖音、小红书等平台，以“校园生活交友平台”为主题，使用 HTML5 + CSS3 + JS 实现模拟的校内社交互动平台，并完成演示答辩与个人设计报告。

## 2. 技术要求
- 主要采用 HTML5、CSS3、JavaScript 实现。
- 兼容主流浏览器，支持不同屏幕尺寸。

## 3. 功能需求（按大作业要求）

### 3.1 首页与游客浏览（模块①，必做）
1) 首页包含网站名称 Logo、导航菜单、最新发布动态流等元素。
2) 游客可浏览注册用户设置为公开的动态，但无法评论与互动。
3) 页面整体视觉风格统一、布局清晰、交互流畅，有一定美感与体验。
4) 可选：首页根据系统数据变化动态更新推荐内容（如热门推荐）。
5) 可选：首页支持简洁动效提示新内容（如动态标记、跑马灯）。

### 3.2 用户系统（模块②，必做）
1) 学号注册，注册流程需输入校验与异常提示。
2) 注册可设置头像、昵称、个人简介、兴趣标签等。
3) 登录流程完整，提供记住我、密码可见、忘记密码交互；登录后进入个人主页查看与编辑信息。
4) 管理员账号支持封禁用户、重置资料等操作。
5) 可选：资料页展示最近活跃时间、访问量、相册墙等。

### 3.3 动态发布与浏览（模块③，必做）
1) 登录用户可在发布页编写并发布图文动态（支持图片上传或链接）。
2) 动态展示在首页时间流中，支持点赞、评论、删除（仅本人）。
3) 动态详情页支持查看全文、点赞数、评论列表与时间戳。
4) 可选：话题标签、可见范围（公开/仅好友）。
5) 可选：编辑或撤回动态，评论区支持表情或图片。

### 3.4 互动与社交（模块④，必做）
1) 注册用户可访问他人主页并浏览其公开动态。
2) 支持关注/取关，主页展示粉丝/关注人数与“TA的动态”。
3) 首页可切换“关注动态”和“全站动态”。
4) 可选：私信（界面展示）、转发、收藏功能。
5) 可选：主题切换（夜间模式）、隐私权限（仅自己可见）。

### 3.5 个性化与创新（模块⑤，可选）
- 推荐系统、搜索功能、用户群组、打卡、年度回顾等。

## 4. 团队与分工要求
- 团队 3-5 人完成，模块①-④的普通功能为必做。
- 每位成员必须独立负责一个模块的前端设计与实现。
- 若为 1-2 人团队，仍需完成四大模块且无额外加分。

## 5. 演示与评分关注点
1) 功能实现完整性（40 分）。
2) 用户体验与交互设计（10 分）。
3) 视觉美感与细节（5 分）。
4) 性能与适配设计（5 分）。
5) 团队协作与贡献（5 分）。
6) 功能创新与亮点（5 分）。

## 6. 报告要求
- 个人独立完成 6-8 页设计报告。
- 内容包含：整体分析与平台设计、个人模块设计与实现、项目总结与反思。
- 第 18 周提交电子版至 Blackboard，纸质版交任课老师。

## 7. 数据与接口约定
- 统一使用 LocalStorage 作为数据层。
- 数据结构与 API 见 `docs/LocalStorage.md`。

## 8. 验收标准
- 模块①-④必做功能可运行，主流程通顺。
- 视觉与交互一致，移动端可用。
- 模块职责清晰，代码可独立运行并可集成。
